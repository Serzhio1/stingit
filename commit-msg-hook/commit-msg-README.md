# commit-msg

## Проблема

Давай рассмотрим примеры нескольких коммит-сообщений

- fix some bugs
- refactoring code
- update versions
- some fixes

Если подобные сообщения не вызывают у тебя много вопросов, и тебе достаточно этой информации, чтобы понять суть изменений в коде, то не трать свое время, этот хук тебе не нужен

## Решение

Если ты читаешь дальше, значит тебе важно, чтобы в твоем проекте сообщения коммитов были качественными, достаточно подробными и соответствовали определенному формату. Нарипример, такому:

```
TYPE [TASK_PREFIX-ID] CHANGE_DESCRIPTION
```

| Часть | Описание |
|-------|----------|
| `TYPE` | тип изменения. Например `fix:`|
| `TASK_PREFIX` | префикс задачи. Например `OSG`|
| `ID` | номер задачи. Например `1731`|
| `CHANGE_DESCRIPTION` | описание изменений |

## Примеры валидных сообщений

```
fix: [OSG-0123] исправлен баг авторизации, связанный с некорректной валидацией токена
```

```
refactor: [ESB-77] переработан модуль расчёта стоимости, уменьшено количество вложенных условий
```

```
test: [CSK-9812] добавлены интеграционные тесты для проверки корректности отмены транзакций
```

## Смогу ли я конфигурировать настройки валидации? Да!

Ты можешь изучить реализацию хука в Stingit, написанного на `bash`, и вдохновиться им, чтобы написать свою собственную версию. Однако, если тебе нравится данная реализация, то с помощью переменных, ты можешь легко и просто настроить хук под себя

| Часть | Что можно изменить |
|-------|----------|
| `TYPE` | Определяй свой набор типов коммитов. Например `fix:`, `test:`, `docs:`, `bug:`|
| `TASK_PREFIX` | Определяй свой префикс задач. Например `AAPL` или `GE` или `NKE`|
| `ID` | Определяй свои ограничения на номер задачи. По-умолчанию - любая непустая последовательсть цифр. Например `00214`|
| `CHANGE_DESCRIPTION` | Определяй свои ограничения по минимальному кол-ву слов и символов в описании изменений|

## Есть сомнения к качеству хука? Это правильно!

В каждой директории рядом с хуком располагаются тесты, которые ты можешь просмотреть, запустить, и убедиться в качестве скрипта
